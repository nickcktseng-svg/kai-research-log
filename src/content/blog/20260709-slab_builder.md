## 📝 專案簡介與動機

為了計算化學的表面模擬需求，我們需要頻繁地從塊材 (Bulk) 結構中切出特定的晶面 (Slab) 並加上真空層。與其每次都手動修改參數與指令，不如開發一款自動化的小工具。

這篇日誌記錄了我如何從零開始，利用 Python 打造出一款內建 3D 視覺化、無須依賴外部軟體 (如 VESTA) 的桌面端計算化學工具。

## 🗺️ 開發大綱與進化史 (Step-by-Step)

### Phase 1：環境建置與最小可行產品 (MVP)
* **目標：** 寫出第一支能成功讀取 POSCAR 並切出 Pt(111) 晶面的 Python 腳本。
* **技術：** Python, ASE (Atomic Simulation Environment) 套件。
* **過程：**
  1. 在 Mac 環境下配置 VS Code 與終端機。
  2. 手動撰寫鉑 (Pt) 的 Bulk 原料檔。
  3. 寫出核心運算邏輯：`ase.build.surface` 切割晶面、擴展 Supercell、輸出 VASP 可讀的 POSCAR 格式。

### Phase 2：從腳本走向圖形介面 (GUI)
* **目標：** 擺脫終端機指令，讓工具變成有按鈕、可輸入參數的軟體。
* **技術：** Tkinter。
* **過程：**
  1. 原本嘗試使用網頁框架 Streamlit，但遇到本機網路連線超時的問題。
  2. 果斷轉換跑道，改用 Python 內建的 Tkinter，成功打造出能在 Mac 桌面上獨立運行的 App 視窗。
  3. 串接了系統原生的「選擇檔案」與「存檔」對話框，大幅提升操作直覺度。

### Phase 3：內建 3D 視覺化 (取代 VESTA)
* **目標：** 在軟體介面右側直接顯示產生的原子結構透視圖。
* **技術：** Matplotlib 3D 繪圖, FigureCanvasTkAgg。
* **過程：**
  1. 抓取 ASE 產生的三維座標，利用散佈圖 (Scatter) 畫出原子。
  2. **視覺優化：** 解決了 Matplotlib 預設會把模型拉長變形的問題，強制鎖定真實物理比例 (`set_box_aspect`)。
  3. **質感升級：** 關閉所有醜陋的預設座標軸與灰色背景，改用 VESTA 風格的乾淨白底與銀白色立體原子。

### Phase 4：導師回饋與專業細節打磨
* **目標：** 提升科學繪圖的嚴謹度與易讀性。
* **過程：**
  1. **標示真空層：** 寫迴圈畫出 Unit Cell 的 12 條外框線，讓真空層的範圍肉眼可見。
  2. **Jmol 官方演色表：** 導入 `ase.data.colors`，讓程式自動判別原子序（如 Pt 為銀白、Cu 為紅銅色），未來遇到混合金屬也能完美顯示。
  3. **XYZ 空間座標軸：** 在左下角建立紅綠藍三色的獨立座標軸，確保旋轉時方向不迷失。

### Phase 5：靈光一閃的終極功能 — 多分頁系統 (Tabs)
* **目標：** 能夠同時開啟多個不同參數的 Slab 進行比較。
* **技術：** ttk.Notebook, 事件監聽器 (Event Listener)。
* **過程：**
  1. 這是我自己發想的功能！將單一畫布升級為多分頁系統。
  2. 寫入滑鼠拖曳監聽器，讓左下角的 XYZ 小座標軸能與主畫面的 3D 模型「完美同步旋轉」。
  3. 同步補齊了自動匯出 `slab_info.txt` 紀錄檔的功能，專案正式 100% 完工。

## 💡 開發過程中的關鍵問題與除錯紀錄 (Q&A)

在這個過程中，遇到了幾個非常經典的程式與科學問題，記錄下來作為未來的避坑指南：

**Q1：在 Mac 終端機輸入安裝指令，卻跳出 `zsh: command not found: pip`？**
> **解法：** Mac 環境下的 Python 3 通常需要明確指定版本，將指令改為 `pip3 install ase`，或是透過 `python3 -m pip install ase` 喚醒安裝程式即可解決。

**Q2：程式讀取 POSCAR 時，跳出 `RuntimeError: The number of scaling factors must be 1 or 3`？**
> **解法：** 這是新手最常犯的「實驗室失誤」——忘記存檔或格式跑版。VS Code 檔名旁邊出現白色小圓點代表未存檔，導致程式讀到舊的錯誤資料。此外，VASP 的 POSCAR 格式非常嚴格，第一行必須是註解，第二行必須是縮放比例數字，不能有多餘的空白行。

**Q3：為什麼 Matplotlib 畫出來的 3D 模型，長寬高比例嚴重失真，原子看起來扁扁的？**
> **解法：** Matplotlib 的 3D 引擎預設會把繪圖區壓縮成一個「正方體」。因為我們的模型加入了很厚的真空層（Z 軸很長），導致 X 和 Y 軸被嚴重擠壓。必須抓取結構的真實長度，並加上 `ax.set_box_aspect((x_max, y_max, z_max))` 來強制鎖定物理比例。

**Q4：為什麼左下角的 XYZ 小座標軸，沒有辦法跟著主畫面一起旋轉？**
> **解法：** 因為它們是兩塊不同的畫布 (Axes)。必須寫一個事件監聽器 `fig.canvas.mpl_connect('motion_notify_event', sync_views)`。當主畫面被滑鼠拖曳時，自動將主畫面的仰角 (elev) 與方位角 (azim) 數值，即時賦值給左下角的小視窗，達成同步連動。

**Q5：Slab 參數中的 Miller index、Layers、Vacuum 和 Supercell 到底代表什麼物理意義？**
> * **Miller index (米勒指標)：** 決定從 Bulk 中切哪一個面，不同面（如 111 或 100）的原子排列緊密度與催化活性完全不同。
> * **Layers (層數)：** 模擬真實材料深度的厚度。太薄會導致上下表面電子互相干擾，太厚會增加計算成本。
> * **Vacuum (真空層)：** 為了打破 VASP 在 Z 軸上的週期性連續邊界條件，必須加入 10-20 Å 的空白區來創造出「表面」。
> * **Supercell (放大倍數)：** 將 XY 平面放大（如 2x2），確保未來放上吸附物分子時，分子與其週期性分身之間有足夠的安全距離，避免不合理的排斥力。

## 專案動機與目標

因應實驗室指導老師的作業要求，我決定建立一個專屬於自己的數位空間。這個平台不僅能公開展示學習歷程，還要具備實用的文獻檢索功能，協助我更系統化地處理化學領域的研究資料。

在評估了多種方式後，我選擇了擴充性強的靜態網站框架，並利用開源資源來打造這個平台。

---

## 階段一：基礎建設 (Astro + Cloudflare Pages)

一開始的挑戰在於環境建立與部署。我選擇了 **Astro** 作為網頁框架，因為它內建了非常適合撰寫研究日誌的內容管理系統（Content Collections）。
![Astro部署流程](/blog/20260628/20260628-1.png)
* **本地端開發：** 透過終端機指令 `npm create astro@latest` 建立基礎架構，並學習使用 Git 將原始碼推送到 GitHub。
* **自動化部署：** 將 GitHub 儲存庫與 **Cloudflare Pages** 連動。現在只要我推送新的 Markdown 筆記，網站就會在幾分鐘內自動更新，完全不需要自己管理伺服器。
* **視覺重塑：** 為了讓閱讀體驗更好，我重寫了 `global.css` 與排版佈局，將原本預設的亮藍色調，改為乾淨、簡約且專注於內容呈現的極簡黑白風格。

---

## 階段二：建立我所需要的工具
我先將自己所需要的工具內容都列出，交給Gemini評估優先順序，避免同時進行太多事情而搞混
![事件安排順序](/blog/20260628/20260628-2.png)
![AI評估後的順序](/blog/20260628/20260628-3.png)

首先評估出來我第一個改善的是，在主頁可以有快速的互動方塊可以連結至Gemini、Github、Cloudflare甚至還有我個人的音樂帳號也可以透過那些互動方塊去點選

我也改善了我的日誌搜尋功能，可以透過日期或是關鍵字，來使我快速找尋我所需要的日誌，例如未來當我又再度需要建設一個網站時，我就可以透過「網站」、「個人」等關鍵字來進行搜尋。

另外是老師所要求的「文獻搜尋系統」
老師特別提醒，要搜尋特定的 paper，單靠 Google 查絕對不夠全面。因此，我在網站中獨立開發了一個 `/literature` 頁面。
* **技術選型：** 為了符合不綁付費 API 的原則，我選擇串接了開源且免費的 **OpenAlex API**。
* **功能實作：** * 實作了即時關鍵字搜尋功能。未來在查找如錯合物滴定（Complexometric Titration）、有機合成或是觸媒相關的文獻時，可以直接在自己的網站上進行。
  * 加入了動態排序功能，能依照「關聯度」、「引用次數」或「年份」快速過濾，並讓系統自動抓取 Abstract 摘要與作者群，大幅省去了一篇篇點開 PDF 的時間。
以上是針對paper搜尋工具的一些改善

最後還有一個PDF的摘要分析工具
目前Gemini給出的辦法都是透過mac開機連接網路來做分析，我評估下來覺得這個成本太大，也不適合用在我身上，所以選擇暫緩
希望後續會有更好的解法，可以提供我做使用

## 階段三：跨域整合與未來規劃

這個網站除了作為實驗室的進度報告，我也規劃了幾個未來的應用方向：

1. **跨域學習存放處：** 除了本科的化學專業，這裡也能作為我經營翻譯專頁、跨域學習的彙整地，讓零散的思緒慢慢變成有系統的第二大腦。
2. **本地端 AI 分析 (Next Step)：** 這個就是在上面所提及，暫時無法完成的內容，雖然現階段先完成了文獻搜尋，但未來計畫嘗試在本地端部署 Ollama 等開源模型，挑戰在不外流資料的前提下，直接讓電腦幫我總結長篇文獻。

這是一次非常扎實的實作經驗，期待這個平台能隨著我的研究進度一起成長，讓這裡面的內容可以變得更加的豐富。
